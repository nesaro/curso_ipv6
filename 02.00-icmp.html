Nuevos usos de icmpv6

  • El tamaño máximo de paquete de una ruta debe descubrirse de antemano con ICMPv6
  • La parte de gestión de grupos multicast de IGMP ha sido incluida en ICMPv6
  • Funcionalidad ARP ha sido incluida en ICMPv6
  • Soporte para Mobile IPv6 ha sido incluido en ICMPv6
  • RFC 2463

Jerarquía de mensajes ICMPv6

  • 0-127:Errores
    • Destination Unreachable (message type 1)
    • Packet Too Big (message type 2)
    • Time Exceeded (message type 3)
    • Parameter Problem (message type 4)
  • 128-255:Información

Tipos de mensaje de destino inalcanzable

  • 0: Sin ruta al destino
  • 1: Comunicación con destino prohibida administrativamente
  • 2: Fuera del ámbito de la dirección de origen
  • 3: Dirección inalcanzable
  • 4: Puerto inalcanzable

Tipos de mensaje de tiempo excedido

  • 0: Numero de saltos agotados
  • 1: tiempo de reensamblaje de paquete agotado

Tipos de mensaje de problema de parámetro

  • 0: Campo erróneo en cabecera
  • 1: Campo próxima cabecera desconocido
  • 2: Opción ipv6 desconocida
  • En todas ellas se identifica el octeto en el que se encontró el error

Mensajes informativos

  • Echo Request, Reply
  • Multicast Listener Query,Report,Done
  • Router y Network Solicitation, Advertisement. Redirect
  • Router Renumbering, ICMP Node Information Query, Response
  • Inverse ND Solicitation, Adv Message
  • Mobile: Home address discovery Request, Reply, Prefix Solicitation, Advertisement format

Estructura de paquetes

  • Siempre precede una cabecera IPv6, con un nextheader 58
  • 2bytes: tipo
  • 1bytes: código
  • 2bytes: Checksum
  • resto: datos (Varían según el tipo)
