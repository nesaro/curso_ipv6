Curso IPv6

Convivencia IPv4-IPv6

La pila dual

  • Hasta que las redes IPv6 se impongan, es necesario un mecanismo de convivencia
  • Vamos a explorar dos alternativas:
    • Túneles y Traductores

Métodos de Tunel

  • Manuales
  • Tunel Broker
  • Automáticos
  • 6to4
  • 6over4
  • ISATAP
  • Teredo
  • Tunel setup protocol
  • Dual Stack Transition Mechanism (DSTM)
  • Open VPN

Túneles manuales

  • Permite comunicar islas IPv6
  • Se configura en el cliente y el servidor
  • Es transparente para las aplicaciones

6to4

  • Permite comunicar islas IPv6
  • No requiere configuración en el cliente
  • Es transparente para las aplicaciones
  • TODO: Como enruta?

6over4

6over4 RFC 2529

  • Permite comunicar islas IPv6
  • No requiere configuración en el cliente
  • Es transparente para las aplicaciones
  • Utiliza direcciones multicast IPV4
  • No escala en redes grandes

Métodos de Traducción

  • NAT-PT
  • Socks

NAT-PT

  • Realiza el trabajo del NAT y traduce también el protocolo
  • No es transparente a las aplicaciones, requiere cambios para peticiones DNS o para FTP (DNS-ALG)

Socks

  • Utiliza una arquitectura cliente servidor
  • El servidor genera una dirección IPv4 falsa y la comunicación se realiza a través de esta
  • En conversión 6-4 utiliza el algoritmo de traducción SIIT
